# blog-traffic-dev — 블로그 유입용 글감 발굴 도메인 스킬

이 파이프라인의 최종 고객은 **블로거(사용자)**이고, 최종 성과 지표는 **블로그 방문자 수**다.
"트렌드가 진짜인가?"는 중간 질문일 뿐이고, 최종 판정 기준은 항상
**"이 소재로 오늘 글을 쓰면 검색 유입이 생기는가?"**다. 키워드·스코어링·리포트 관련
코드를 만지기 전에 이 문서를 먼저 보라. 실행 설계서: `docs/redesign-blog-traffic.md`.

## 1. 황금 키워드 공식 (모든 판단의 축)

```
기회 = 검색 수요(뜨는 중) ÷ 콘텐츠 공급(이미 쓰인 블로그 글 수)
```

- **수요↑ 공급↓ = 황금 키워드.** 신생 키워드는 경쟁 문서가 적어 신규 블로그도 상위 노출이 쉽다.
  방송 직후·신메뉴 출시 직후의 좁은 시간창이 바로 이 상태다.
- 수요만 높고 공급도 많은 키워드(예: "성수 맛집")는 대형 블로그가 점유 — 글 써도 묻힌다. 제외 대상.
- 공급 지표는 네이버 블로그 검색 API의 `total`(문서 수)로 측정한다. 수요 지표는 데이터랩
  상승률/burstRatio. **둘 중 하나만 보고 판단하지 마라.**

## 2. 키워드 검증 절차 (어디서 어떻게 검색하는가)

추출된 키워드가 진짜 글감인지 확인하는 표준 절차. 자동화(STEP 4.5)와 사람 검증 모두 이 순서다:

1. **네이버 통합검색** (search.naver.com) — 키워드를 그대로 검색. **그 의미로 결과가 나오는가?**
   동음이의 오염 체크: "콘크리트" 검색 → 건축자재만 나옴 = 검색 불가 키워드.
2. **네이버 블로그탭 + 최신순/기간 필터(1주)** — 이미 쓰인 글 수 = 경쟁도(공급).
   최근 1주 글이 0~수십 건이면 기회, 수백 건 이상이면 늦었거나 레드오션.
3. **네이버 데이터랩** (datalab.naver.com/keyword/trendSearch) — 검색량 **추이**(상승 중인가).
   주의: 수치는 그룹 내 상대값이라 절대 비교 불가(trendleading-dev §5).
4. **네이버 검색광고 키워드도구** (searchad.naver.com → 도구 > 키워드 도구, 무료 가입) —
   **월간 검색수 절대값** + 연관키워드 목록. 블로거의 실전 핵심 도구. 여기서 나온 연관키워드가
   글 제목·본문에 넣을 보조 키워드가 된다.
5. (보조) 구글 트렌드 — 네이버 밖 수요 교차 확인.

리포트의 모든 키워드에는 사용자가 1번을 바로 할 수 있도록 **네이버 검색 링크와 근거 기사 URL**을 붙인다.

## 3. 검색가능형 키워드 규칙 (콘크리트 사건 — 재발 금지)

**사건**: '콘크리트 (디저트)'가 키워드로 보고됨 → 사용자가 검색했으나 건축자재만 나옴.
실체는 **버거킹 신메뉴**(바닐라 커스터드 기반 시그니처 디저트). 원인: 여러 브랜드가 나열된
뉴스 요약기사("[오늘 뭐 먹지] CJ프레시웨이·해태제과·농심 외…")에서 브랜드와 메뉴의 연결이
끊긴 채 메뉴명만 추출됨. 올바른 키워드는 **"버거킹 콘크리트"**였다.

규칙 (프롬프트와 검증 코드 양쪽에 적용):
1. 키워드는 **단독 검색으로 의미가 통하는 형태**여야 한다:
   - 브랜드+메뉴: "버거킹 콘크리트" (O) / "콘크리트" (X)
   - 지역+가게명: "성수 설이동" (O) / "설이동"만으로 검색이 되면 단독 허용
   - 프로그램+소재: "언더커버 쉐프 식당" (O)
2. **일반명사·동음이의어 단독 금지**: 콘크리트, 테이크(take), 크라운 등.
3. 원문에서 브랜드/지역/출처를 못 찾으면 키워드를 버리거나 confidence를 낮춰라 —
   **추측으로 브랜드를 붙이는 것은 더 나쁘다**(오보 리포트).
4. **뉴스 요약기사 함정**: 제목에 "[오늘 뭐 먹지]", "외", "·"로 여러 브랜드가 나열된 기사는
   본문 스니펫이 잘려 브랜드↔메뉴 매칭이 자주 끊긴다. 이런 기사發 키워드는 근거(reason)에
   브랜드가 명시되지 않으면 신뢰하지 마라.
5. 유사 변형('우베 디저트'/'우베 (ube)'/'우베 (ube) 디저트')은 하나로 병합 — 배치 내
   `mergeSimilarKeywords`만 믿지 말고 DB 누적분과의 대조도 고려하라.

## 4. 반복 키워드 = 신뢰 붕괴

같은 키워드가 매일 리포트에 나오면 사용자는 리포트를 안 읽게 된다. 반복의 4대 원인과 방어:
1. 같은 기사 재수집 → 크로스데이 URL dedup (설계서 P0-1)
2. 오래된 글 유입 → 발행일 3일 필터 (P0-2)
3. LLM이 매일 같은 걸 "발견" → 기추출 키워드를 프롬프트 제외 목록으로 주입 (P0-3)
4. 탐침 풀 하드코딩 키워드('설이동' 등)가 매일 stats 적재 → 탐침發 키워드는 리포트에서
   "탐침 감지"임을 표시하고, 다이제스트 상위 노출엔 쿨다운 적용 (P3-3)

**연속 노출이 항상 나쁜 건 아니다**: 진짜 지속 상승 트렌드는 "관찰 중" 섹션에 분리 표기한다.
새 소재(황금 소재)와 지속 트렌드를 같은 자리에 섞지 마라.

## 5. 실증된 성공 소재 패턴

| 유형 | 실증 사례 | 성공 조건 | 글 각도 |
|---|---|---|---|
| **방송미디어發** | '언더커버 쉐프 식당' 글 조회수 급증 (사용자 실증, 2026-07) | 방송 직후 24~48시간 내 발행. 시청자가 "그 식당 어디?"를 검색하는 시간창 | 위치/예약/메뉴/가격 정리형 — 검색 의도가 "정보 찾기"이므로 정보 완결성이 핵심 |
| 신메뉴출시發 | 버거킹 콘크리트 유형 | 출시 발표~판매 첫 주. 공식 정보가 아직 흩어져 있을 때 | 실제 후기 + 가격/칼로리/비교 |
| 지역맛집發 | 오픈런/웨이팅 신상 매장 | 오픈 초기, 블로그 글 수십 건 이하일 때 | 웨이팅 팁/솔직 후기 |
| 시즌성 | 매년 반복 (수박 디저트 등) | 시즌 시작 2주 전 선점 | 올해 신상 큐레이션 |

- **방송미디어發이 최우선 순위** — 실증됐고, 수요 폭발+공급 공백의 시간창이 가장 뚜렷하다.
  수집 쿼리에 방송 시그널("방송에 나온 맛집" 류)을 유지하라 (P3-2).
- 시즌성은 Critic이 "매년 반복"으로 REJECT하기 쉬우나, 블로그 유입 관점에선 **시즌 초입 선점**은
  유효한 글감이다. content_type으로 구분해 버리지 말고 표기하라.

## 6. 리포트 품질 기준 (다이제스트 = 제품)

리포트의 각 소재는 **"지금 바로 글을 쓸 수 있는 상태"**로 제공한다. 필수 요소:
1. **공략 키워드** (검색가능형, §3 규칙 통과)
2. **왜 지금인가** (수요 근거: 급등/방송/출시 + 공급 근거: 블로그 글 n건뿐)
3. **경쟁도** (블로그 문서 수)
4. **근거 링크** (네이버 검색 링크 + 원본 기사 URL — 사용자가 30초 안에 팩트체크 가능해야 함)
5. **소재 유형 태그** (§5의 4유형)

이 중 하나라도 빠진 항목을 상위 섹션에 올리지 마라. 검증 안 된 키워드는 "검증 필요" 섹션으로.

## 7. 의사결정 히스토리 (다시 논쟁 말 것)

| 결정 | 이유 |
|---|---|
| 목표 = 블로그 유입 (트렌드 감지는 수단) | 사용자 실사용 목적. '언더커버 쉐프' 실증 (2026-07-09 재설계) |
| 공급(경쟁 문서 수) 지표 도입 | 수요만으론 글감 가치 판단 불가 — 황금 키워드 공식 |
| 검색가능형 키워드 강제 | 콘크리트 사건 — 검색 안 되는 키워드는 리포트 가치 0 |
| UNIQUE 인덱스 대신 코드 레벨 dedup | 기존 DB에 중복 데이터 존재 — 인덱스 생성 실패 |
| 탐침 풀 자동 삭제 금지 | 풀 구성은 사용자 결정 사항 (로그 보고만) |
